# 猫咪智能饮水监测系统 (IoT-Cat-Water-System)

##  项目简介  
本项目是一款基于 **Python Flask** 和 **ECharts** 的智能 IoT 模拟系统。系统模拟了多个猫咪饮水监测设备上报数据到云端服务器的过程，实现了多猫档案隔离、科学饮水判定、异常数据红线预警、断网续传及高并发处理等核心功能。

---

##  核心亮点（优档技术标准）

- **多猫档案独立管理**：通过唯一 `cat_id` 实现数据物理隔离存储，每只猫拥有独立的 `.json` 历史档案。
- **科学阈值与红线预警**：内置科学算法计算正常饮水区间。当数据异常时，看板折线由绿变红，实现直观预警。
- **工业级容错逻辑**：客户端集成指数退避重试算法。若服务器宕机，数据自动存入本地缓存并在网络恢复后自动补传。
- **高性能并发处理**：经 ApiPost 实测，系统具备极高的吞吐量，能够支撑大规模设备上报。
- **数据离线补传**：客户端集成指数退避重试算法，若服务器宕机，数据自动存入本地缓存并在网络恢复后自动补传。
- **密码**：apipost设置密钥正确时，才会传送数据

## 完整实验与测试步骤

### 1. 启动后端监测中心  
服务器将开启 API 接口并自动初始化数据目录 `server_data`。

```bash
# 终端 1
python server.py
```

验证：浏览器访问 [http://127.0.0.1:5000/dashboard](http://127.0.0.1:5000/dashboard) 查看监控看板。

---

### 2. 模拟器单猫连续采集测试（本地模式）  
模拟一台 IoT 设备绑定特定猫咪，连续采集 5 次数据。

```bash
# 终端 2
python run_headless_pro.py
```

---

### 3. Mock 测试（前后端解耦演示）  

> 这是本项目展示“优”档水平的关键步骤，演示在不依赖本地服务器的情况下完成联调。

**切换方法**：打开 `run_headless_pro.py`，将 `API_URL` 切换为 Mock 地址：

```python
# 启用以下 Mock 地址（删掉前面的 # 号）
API_URL = "https://mock.apipost.net/mock/5ced3c80a451000/upload_data?apipost_id=ed3eaf4fcf003"

# 注释掉本地地址（在前面加上 # 号）
# API_URL = "http://127.0.0.1:5000/upload_data"
```

 执行：关闭本地 `server.py`，运行模拟器。此时模拟器将直接从 ApiPost 云端获取动态生成的响应数据（如“达标/不达标”）。

---

### 4. ApiPost 并发压力测试报告  

本项目通过 ApiPost 进行了极限性能测试，实测数据如下：

- **测试场景**：20 客户端持续并发请求  
- **核心指标**：
  - **成功率**：100%（总请求 26,754 次，失败 0）
  - **吞吐量 (TPS)**：891.76 req/s
  - **响应延迟**：平均仅 **9.729 ms**，极大确保了数据上报的实时性

## 项目结构说明

- `server.py`：后端主程序，含 Flask 路由与可视化看板  
- `run_headless.py`：高级模拟器（含单猫锁定、重试、缓存、Mock 切换逻辑）  
- `core/water_recommender.py`：基于猫咪品种与年龄的科学算法核心库  
- `server_data/`：(自动生成) 存放各猫咪独立 ID 的 JSON 档案  
- `pending_uploads.json`：(自动生成) 存放离线待补传的缓存数据
